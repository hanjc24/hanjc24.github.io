LHY-L6 Transformer

Encoder-Decoder架构

Encoder

输入一排向量，输出一排向量

Block=Residual Connection + layer normalization + fully connected network + layer normalization
其中Residual Connection是self attention的输出加上对应原有的输入向量
FC layer=fully connected network=Feed Forward=FF
layer normalization是将一个输出的向量的每一个数值，减去这个向量的所有值的平均值然后再除以标准差。这样每个向量的方差就都是1，均值为0

Decoder: Auto regressive(AT)

输入为Encoder输出的一排向量，以及首先输入的special token BOS(Begin token)。它首先会输出一个vector，维数为Vocabulary size，e.g.语言中表意单位的数目(在中文中或许就是常用字)，然后再选择数值最高的那一位对应的onehot vector对应的字作为第一个最终输出。
然后再把这个输出作为输入，和BOS一起作为Decoder的输入，然后再输出第二个输出，再把这个输出加到输入里。。。以此类推，直到输出为END(有点像RNN)

AT架构

  • 第一层：Masked Multi-head Self-attention + Add&Norm
    Masked指的是计算注意力矩阵的时候只和它左/前侧的向量计算注意力。这是由AT逐步生成生成输出的方式所决定的。因为它右侧的向量都没有意义。每次循环会新增加一个Decoder的输出作为一个新的输入，然后计算得到一个新的输出，其余的输出都和原来保持不变
  • 第二层：Multi-head Self-attention + Add&Norm
    这是连接Encoder与Decoder的桥梁,用到了交叉注意力机制
    实际上就是Decoder第一层的输出提供Query，Encoder的输出提供key与value，进行cross attention计算匹配度并对v进行加权求和，抽取有关联的输入资讯作为输出给
    第三层。
    实测来看，Decoder的注意力的峰值也是在从左向右移动的
  • 第三层：FF + Add&Norm

补充：NAT架构

输入为N个Begin，输出为N个向量，如果其中有END则舍弃其右侧的字符。N可以用另一个神经网络来决定，它的输入为encoder的输出，输出为一个数值N。
这样做的好处就是可以并行计算，NAT的每个输出可以并行处理
NAT高速低延迟但输出的token质量难保证，不能很好地处理输出token之间的逻辑关系

note: 现在decoder-only的LLM用的都是AT啦

Training Transformer

数据处理

一串输入向量，标注好输出的一串向量。
e.g.一段声音信号，标注好Label(预期的输出)

训练的目标就是让Decoder的输出越接近正确答案越好。记得输出的一串向量后面也一定要加上END，这也是预期的输出。
但是由于Decoder的输入就是它应当输出的正确答案，所以要用一种掩码技术

Guided attention

要求机器在做attention的时候一定要看过每一个输入，在attention的时候有固定的过程。（用于语音合成、语音识别、翻译等）
也就是让attention的峰值必须由左向右

Beam Search

局部最优并非全局最优.可以通过Beam Search找到较优解
适用的任务：语音辨识、语音合成（有明确正确答案或评分标准，分数直接反映答案的好坏）
不太适用：需要一些创造力的工作（譬如完成句子，有些随机性有时会更好，分数最高的答案不一定最好；譬如TTS，语音合成，可以在Decoder加noise,测试的时候加noise反而结果更好）

Scheduled Sampling

在训练中就可以给Decoder加一点错误的输入，增强其处理输出错误后回正的能力（避免一步错步步错）
