DeepSeekNSA

要点

  • 三种KV映射方式，一层三次attention计算，blockwise select与trainable参数选择模式
  • 计算时使用GQA+query group共用相同的selected blocks，保证计算效率充分利用GPU，slc策略的attention计算中，每次加载一个query group同位置ttt上的所有qtq_tqt​

Arithmetic Intensity

  • Arithmetic Intensity =compute operatioonsmemory access= \frac{compute\ operatioons}{memory\ access}=memory accesscompute operatioons​由算法任务决定
  • GPU’s Critical Arithmetic Intensity =peak compute capabilitymemory bandwith= \frac{peak\ compute\ capability}{memory\ bandwith}=memory bandwithpeak compute capability​由GPU硬件本身决定
  • 前者大于后者：compute bound(limited by GPU FLOPS)
    前者小于后者：memory bound(limited by memory bandwith)

普通LLM：训练与推理prefill阶段展现为 compute bound(计算密集的大矩阵乘法)，而decode展现为 memory bound(单一query进行自回归生成新一个token)
优化目标是减少training&prefill的计算开销与decode时的访存开销。

  • 训练与prefill计算开销减少：通过稀疏选取和压缩注意力实现，由稀疏度保证；
    • 这个效果被具体策略最大化：Blockwise选择与GQA；KV-cache transfer的精心设计
  • decode访存优化：
    • NSA：每层需要的KV条数=sd+nl′+w=\frac{s}{d}+nl'+w=ds​+nl′+w，其中s为序列长度，n为选取块数，l’为slc块大小，d为cmp块首间距。随着s线性增长，但是常数为1d\frac{1}{d}d1​
    • Full: 每层需要的KV条数为sss

ot=Attention(qt,Kt~,Vt~)o_t=Attention(q_t,\tilde{K_t},\tilde{V_t}) ot​=Attention(qt​,Kt​~​,Vt​~​)

映射选取的综合考虑：

  • 参数选取，是否training-free还是要有可训参数
    • 不可训如何保证模型能力不受损(根据qtq_tqt​ semantic meaning Dynamic还是人为规定)
    • 要训的话如何保证有梯度，反传如何保证训练效率

算法设计

主要解决的问题：要想达到FlashAttention-level的加速，且实现sparse attention的话，不能像FlashAttention每次加载连续的query block(多个不同位置的query向量)，因为它们的sparse select blocks不一致。
所以使用GQA+query group共用相同的selected blocks
