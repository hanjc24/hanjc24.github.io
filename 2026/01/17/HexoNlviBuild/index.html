How to customize Hexo theme Nlvi

缘起

Nlvi是我的一位学长在他的blog魔改并使用的主题，我也装了Nlvi试了试，效果不错，尤其是Tag的索引做得挺清楚的，整体很简约。

唯一的美中不足是原本的Nlvi主题配色的底色都是浅色，观感上过于明亮，所以我研究了一下Hexo theme的结构，修改了一些生成css部分的代码。

参考文档

  • Hexo official docs
  • Hexo的安装与配置教程
  • theme Nlvi git

Hexo的安装已经有很详尽的文档了，此处不再赘述
在安装主题的时候，一定不要把Nlvi(小写L)打成NIvi(大写I)，尤其是在vscode边栏目录中这个区别很难分辨，导致hexo g的时候会报找不到layout的错误
除此之外记得把themes/Nlvi下的_config.template.yml拷贝到blog根目录下并改名为_config.Nlvi.yml

Hexo工作逻辑

hexo生成预制博客的方式主要是：

graph LR
    A[加载配置
_config.yml
_config.Nlvi.yml] 
    A --> C[加载源文件
\source下的post等];
    A --> D[加载主题文件
\themes\Nlvi];
    C --> E[渲染源文件];
    D --> F[模板引擎];
    F -- 布局 --> G
    E --嵌入--> G[生成完整html]

加载配置时，首先加载的是_config.yml，然后根据其中theme: Nlvi字段在blog根目录下找到_config.Nlvi.yml进行加载。
对同一字段的配置有不同时，前者的配置会覆盖后者。

添加、修改文章

hexo官方提供的做法是

创建post文章:

title为text1,使用默认layout模板post(这个模板文件是\scaffolds\post.md)
这个命令的作用仅仅是在\source_posts路径下安装上述模板创建一个名为text1.md的文件
也可以使用不同的模板、创建自己的模板或者直接在vscode中新建文章。

hexo自动生成了Front-matter，它是文件开头的 YAML 或 JSON 代码块，用于配置文章。

修改主题配色

总体方法

graph TD
    A[在html右键检查
想修改的元素] --> B
    B[找到这个元素对应
html的class或id等] --> C
    C[在Nlvi目录下的
\source\style目录
检索找到的class名] --> D
    D[自由发挥修改color字段]

除此之外，有必要的话还可以检索生成的style.css(位于/public/style路径)

实战

修改header的hover颜色

  • 首先检查元素
    找到其class为menu-item
  • 检索style.css 注意到两个color字段，一个是hover时的颜色(#89cce7)，一个是正常颜色(inherit继承自.header)
  • 检索styl文件
    找到header.styl
    路径：themes\Nlvi\source\style\_partial\_schemes\_banderole\header.styl $theme-color-hover实际上是这个脚本语言定义的变量，会被模板引擎解析并填充以生成最终的css
    可以直接在此修改为固定的颜色，也可以尝试修改这个变量的计算逻辑
  • 重新hexo g生成页面

总结

Nlvi主题预设的styl脚本变量都在：\themes\Nlvi\source\style\_common\variable.styl这个文件内
它会根据_config.Nlvi.yaml的theme-style:字段来自动选择主题色，并mix混色生成所需的颜色。
所以也可以自由发挥修改这个文件中对各类颜色的预设与计算方式，以达到修改主题颜色的目的。
